# Entrega ao cliente

## Objetivo

Padronizar a entrega final de projetos ao cliente, garantindo clareza sobre publicação, acessos, garantia e próximos passos.

## Quando usar

Use após publicação final aprovada e antes de arquivar o projeto internamente.

## Pré-requisitos

- Site publicado e validado.
- Aprovação final registrada.
- Pagamento final confirmado quando aplicável.
- Recibo ou comprovante preparado.
- Lista de acessos e links organizada.

## Passo a passo

1. Confirme aprovação final do cliente por escrito.
2. Confirme se há saldo pendente ou pagamento final.
3. Emita recibo ou documento combinado.
4. Envie o link final do site publicado.
5. Informe o domínio publicado.
6. Explique o prazo de garantia combinado.
7. Liste o que está incluso após a entrega.
8. Liste o que não está incluso, como novas páginas, redesign, campanhas ou integrações não contratadas.
9. Entregue acessos quando aplicável: GitHub, Cloudflare, Analytics, Search Console, GTM, Clarity ou outros.
10. Oriente o cliente a guardar acessos de domínio e DNS.
11. Registre a entrega no Notion/CRM.
12. Arquive o projeto no local interno correto.
13. Agende follow-up se houver suporte, manutenção ou upsell.

## Como validar

1. Cliente confirmou recebimento dos links.
2. Cliente entende prazo de garantia.
3. Todos os acessos aplicáveis foram entregues ou registrados.
4. Projeto está registrado como entregue no controle interno.
5. Recibo foi enviado quando aplicável.

## Problemas comuns

- **Cliente pede mudança fora do escopo:** registrar como nova demanda ou orçamento.
- **Cliente não sabe acessar domínio:** orientar sem assumir propriedade indevida.
- **Pagamento pendente:** alinhar antes da transferência final de materiais quando combinado.
- **Acesso em conta errada:** corrigir permissões e documentar.
- **Garantia mal entendida:** explicar que garantia cobre correções do entregue, não novas funcionalidades.

## Checklist final

- [ ] Aprovação final confirmada.
- [ ] Pagamento final recebido, se aplicável.
- [ ] Recibo emitido.
- [ ] Links finais enviados.
- [ ] Domínio publicado informado.
- [ ] Prazo de garantia informado.
- [ ] Inclusões e exclusões pós-entrega explicadas.
- [ ] Acessos entregues, se aplicável.
- [ ] Projeto registrado no Notion/CRM.
- [ ] Projeto arquivado.
