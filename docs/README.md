# Documentação operacional da Maximos Lab

Esta área reúne runbooks e checklists internos para padronizar publicação, configuração técnica, validação e entrega de projetos da Maximos Lab.

A documentação foi pensada para ser usada durante projetos reais de landing pages, sites institucionais, portfólios, templates e páginas publicadas com GitHub Pages, Cloudflare e domínio próprio.

## Como usar

1. Abra o runbook correspondente à etapa atual do projeto.
2. Siga o passo a passo na ordem indicada.
3. Use a seção **Como validar** antes de avançar.
4. Registre qualquer exceção no controle interno do projeto.
5. Antes da entrega, execute os checklists finais.

## Regras internas

- Nunca registrar domínios em contas pessoais da Maximos Lab quando o domínio pertence ao cliente.
- Nunca inserir senhas, tokens, chaves de API ou dados privados nos arquivos do repositório.
- Sempre validar HTTPS, DNS, SEO básico, responsividade e links antes da entrega.
- Sempre documentar exceções combinadas com o cliente.
